React-Native Mobile App for Replay FX Convention



This is a mobile app built for the ReplayFX Convention that takes place every summer at the Pittsburgh Convention Center in late July. Last year the convention was attended by 10,000 people. This convention is about gaming and is the home of the Pinburgh Championships with prize money of $100,000. The convention center will be filled with pinball and arcade games on free play, as well as many other activities.


In addition to being a scheduling app, it lists the arcade and pinball games. It also displays the schedule filtered by the type of event, such as competition, music, movie, etc. Another feature of the app is that it allows users to select "favorites" among the events to add them to their own schedule of favorites. Push Notifications fire 15 minutes before the favorited events.


The app is currently available on Google Play and the App Store.

ASP.net, C# MVC Back End for Replay FX Convention



This was a very good, practical experience for me to add features to a codebase, which is primarily what a developer would be doing as an employee. I added a couple of tables to an existing database. There already was an MVC framework for tables related to the events that will be occuring during the Replay FX Convention. These allow the folks at Replay to create a database of their games to be sent to an API. Topical Tweet Display



As a code challenge to be completed in 72 hours for a development position, I had to use microservices to provide a web-based user interface that lets users view ten recent tweets from one of three topics and RESTful API end points to pull from a database.


I chose to use Sinatra with the Twitter gem and SQLite in development/ Postgres in production for the backend. Because of the requirement of the challenge, the tweets from Twitter have to be stored in a database. If I were really doing this project, I would just use the Twitter API in the front end. The front end is written in React, uses Bootstrap and Sass, Axios, moment, and webpack. The backend is deployed in Heroku and the front end is hosted on AWS.
